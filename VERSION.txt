ThreadGens 0.3.1-runtime-hardening

Branch: fix/runtime-hardening

Included in this version:
- adds CheckedRunner as the safe CLI/script entrypoint
- points Gradle and runnable jars at CheckedRunner
- keeps RedditScreenshotGenerator available as the raw renderer entrypoint
- makes the GUI use checked generation and report missing-output failures
- updates the Windows AI runner to stop on checked generation errors
- updates batch video creation to call CheckedRunner
- refreshes README coverage for Kokoro, Piper, video, batch mode, and profile generation
- expands defaults.txt to document local AI, TTS, and video defaults
- deletes expected output artifacts before each checked run so stale files cannot false-pass
- fails fast when video output is requested while TTS is disabled
- normalizes --tts and --voice-dir before --voice so CLI voice selection is order-forgiving through CheckedRunner

Still included from 0.3.0:
- keeps Piper TTS fully functional as the fallback engine
- adds Kokoro TTS through tools/kokoro_tts.py
- uses the user prompt as the original post
- generates AI replies/comments after the original post
- generates all text first, then all PNG images, then all WAV audio, then all MP4 clips, then the final stitched video
- requests Ollama model unload after script generation by default
- keeps reply views below original post views
- keeps likes below views
- keeps final stitched MP4 output with transitions

Default output folders:
- generated script: output/script/generated_comments.txt
- images: output/
- audio: output/audio/
- video clips and final video: output/video/
